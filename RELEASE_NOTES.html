What's New in 26.06.104

  • A more colorful, lively UI — category-colored icon tiles across the Actions, Settings, Input Source and About pages, colorful sidebar icons, a brighter Caps keycap, glass trigger keycaps, and subtle aurora backgrounds on sheets.
  • "Follow System" language — a new language option that automatically tracks your macOS system language; it's the default for new installs.
  • Fixed high idle CPU usage — the running-status dot animated continuously, keeping the window redrawing every frame and using ~15–20% CPU even when idle. The indicator is now static (still green when running, orange when paused).

26.06.104 更新内容

  • 更鲜亮、更有活力的界面 —— 动作、设置、输入法、关于等页面采用按类别上色的图标方块，彩色侧边栏图标，更亮的 Caps 键帽，玻璃质感的触发键帽，以及弹窗上柔和的极光背景。
  • 「跟随系统」语言 —— 新增可自动跟随 macOS 系统语言的选项，并作为新安装用户的默认设置。
  • 修复空闲时 CPU 占用过高 —— 运行状态指示点的动画会持续运行，使窗口每帧重绘，空闲时也占用约 15–20% CPU。现已改为静态指示点（运行时为绿色、暂停时为橙色）。

What's New in 26.06.103

  • New ways to see your mappings — switch (top-right) between a Grouped by trigger board with 3D keycaps and a full Keyboard map on an Apple Magic Keyboard layout. Click any key to edit it, or an empty key to add a Caps+key mapping. Mappings is now the main screen in a wider window, and "General" is renamed "Settings".
  • AnyDrag integration — HyperCapslock can broadcast the CapsLock-hold so AnyDrag can drag windows while you hold CapsLock, with no extra keyboard hook. Opt-in under Settings (off by default).
  • Push-to-talk (Hold Modifier action) — a new action that holds one modifier (Fn or ⌘⌥⌃⇧) for exactly as long as you hold the trigger key — hold to talk, release to stop.
  • Reopen a collapsed sidebar — added a toggle to bring the sidebar back after hiding it.
  • Polish — clearer trigger-picker labels; refined mapping rows, tooltips and key icons; the hold-modifier HUD now stays up for the whole hold.

26.06.103 更新内容

  • 全新的映射查看方式 —— 右上角可切换「按触发键分组」（3D 键帽）与「键盘布局图」（完整 Apple 妙控键盘）两种视图。点击按键即可编辑，点击空白键即可新增 Caps+键 映射。映射现在是主界面、窗口更宽，原「通用」改名为「设置」。
  • AnyDrag 联动 —— HyperCapslock 可广播 CapsLock 长按状态，让 AnyDrag 在你按住 CapsLock 时拖拽窗口，无需额外键盘钩子。在「设置」中手动开启（默认关闭）。
  • 按住说话（按住修饰键动作） —— 新增动作类型：在你按住触发键期间持续按住一个修饰键（Fn 或 ⌘⌥⌃⇧），按住即说话、松开即停止。
  • 可重新展开侧边栏 —— 新增切换按钮，折叠后可再次打开侧边栏。
  • 细节优化 —— 触发键选择器标签更清晰；映射行、提示气泡与按键图标更精致；按住修饰键时 HUD 会全程显示。

What's New in 26.05.101

  • Inline action setup in the mapping editor — pick Jump, Command, Key Combo, or Open App right from the action dropdown and configure it on the spot (new "Action Type" section), like Switch Input Source. No need to create a separate custom action first.
  • Key Combo actions now compose with held modifiers — holding extra modifiers (e.g. Caps+Option+key) carries into the Key Combo, so the gesture behaves consistently across action types.
  • Fixed a crash when deleting a per-app rule.
  • More resilient config loading — a config with an entry this build doesn't recognize now loads everything else instead of appearing empty, and is auto-backed-up (named by content hash) whenever it can't be fully parsed.

26.05.101 更新内容

  • 映射编辑器内联配置动作 — 直接从动作下拉里选择「跳转 / 命令 / 组合键 / 打开应用」并就地配置（新增"动作类型"分组），和「切换输入法」一样，无需先单独创建自定义动作。
  • 组合键动作支持叠加按住的修饰键 — 额外按住的修饰键（如 Caps+Option+键）会一起作用到组合键上，同一手势在不同动作类型下行为一致。
  • 修复删除按 App 规则时的崩溃。
  • 配置加载更健壮 — 打开包含本版本不认识条目的配置时会正常加载其余内容（而非显示空列表），并在无法完整解析时自动按内容哈希备份配置。

What's New in 26.05.26

  • First CalVer release — version scheme switched from SemVer (last was v2.1.0) to CalVer (YY.MM.build); the monotonic build-number guarantee Sparkle relies on is unchanged.

26.05.26 更新内容

  • 切换到 CalVer 版本号 — 版本号从 SemVer（上一版 v2.1.0）改为 CalVer（YY.MM.build），Sparkle 依赖的构建号单调递增不受影响。
