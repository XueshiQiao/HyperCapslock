三个方案 · 同一条「Caps+Q（默认 = 什么都不做，3 条 App 规则）」

3 条规则：Cursor→分屏（key combo，最高优先级）、iTerm→tmux 分屏（command）、Safari→后退（key combo）。即「各 App 动作不同」的最难情形。

方案 A推荐

代表动作

分屏 3
  • 显示最高优先级那条规则的真实动作，一眼就懂
  • 全部规则同动作时自动收敛成那一个（见下方）
  • 复用现有 pill + 🪟N 徽章，改动最小
  • 各 App 动作不同时 pill 只展示其一，靠 🪟3 提示「还有」
小问题：要不要在 pill 里再加个 App 角标？ 分屏 （vs 仅靠 🪟3 徽章）
方案 B

App 视角

Cursor · 分屏 +2
  • 同时告诉你「在哪个 App、做什么」
  • App 一多就挤；图标加载/缺失要处理，改动更重
  • 和 🪟N 徽章信息重复
方案 C

通用标签

按 App 规则 · 3
  • 永远没有歧义，最简单
  • 只说「是 App 规则」，不告诉你具体做什么
  • 统计页里几乎和「什么都不做」一样无感
方案 A 的收敛：当 3 条规则其实是「同一个动作」时（例：Cmd+W 只在 Terminal / iTerm / Ghostty 启用）
现状 什么都不做 ← 三个终端里都按 Cmd+W 关闭，却显示「什么都不做」
方案 A 关闭 ⌘W 3 ← 动作一致 → 干净地显示那一个动作
