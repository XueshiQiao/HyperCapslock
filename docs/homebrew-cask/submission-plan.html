Getting into the official Homebrew Cask

The cask is technically ready today; the only thing standing between it and brew install --cask hypercapslock is a popularity bar — and we're far below it.

1 · The one blocker: notability

Homebrew Cask only accepts software above a popularity threshold (see its Acceptable Casks policy). The bar is doubled for self-submissions — a PR opened by the owner of the upstream repository, which is exactly our case.

SubmitterThreshold (any one suffices)HyperCapslock nowPass?

Author (us) ≥ 225 stars / ≥ 90 forks / ≥ 90 watchers 7 / 0 / 0 NO
Figure 1. Scale of the gap, not just the numbers: 7 of 225 stars is ~3% of the way there. Forks/watchers are a faster path (90 each) but still untouched.
Reality check

Opening the PR now gets it auto-flagged and closed as "not notable enough". This does not affect distribution — the third-party tap already installs the app. The official cask is discoverability, not capability.

2 · Strategy: prepare & park, then grow

Everything Homebrew checks other than notability is already satisfied: a Developer ID-signed + notarized DMG (passes Gatekeeper), a stable versioned download URL, a homepage, and GPL-3.0 source that ships official binaries — so it correctly belongs in cask, not homebrew/core. The cask file is written and parked in the repo; the work that remains is entirely about growing the audience.

Figure 2. The gate is a single loop: keep promoting until any one metric clears, then run the mechanical submission pipeline once.

3 · The parked cask

Stored at docs/homebrew-cask/hypercapslock.rb. Cleaned to Homebrew idiom vs. the third-party tap: a single url/sha256 (the DMG is universal — no on_arm/on_intel split), a style-compliant desc, a :github_latest livecheck, a macOS floor, and a zap stanza whose paths were verified against the real on-disk files.

cask "hypercapslock" do
  version "26.06.106"
  sha256 "3753…eb3e"

  url "https://github.com/XueshiQiao/HyperCapslock/releases/download/v#{version}/HyperCapslock.dmg"
  name "HyperCapslock"
  desc "Remaps Caps Lock to a Hyper key for vim-style navigation and shortcuts"
  homepage "https://github.com/XueshiQiao/HyperCapslock"

  livecheck do
    url :url
    strategy :github_latest
  end

  auto_updates true
  depends_on macos: ">= :sonoma"

  app "HyperCapslock.app"

  zap trash: [
    "~/Library/Application Support/me.xueshi.hypercapslock",
    "~/Library/Caches/me.xueshi.hypercapslock",
    "~/Library/HTTPStorages/me.xueshi.hypercapslock",
    "~/Library/HTTPStorages/me.xueshi.hypercapslock.binarycookies",
    "~/Library/Preferences/me.xueshi.hypercapslock.plist",
  ]
end

4 · Submission pipeline (run once the bar is cleared)

Figure 3. All mechanical and local-verifiable before the PR — Homebrew CI then re-runs audit/style/install on its own runners.
# 2) generate (downloads, computes sha256, opens editor)
brew create --cask \
  https://github.com/XueshiQiao/HyperCapslock/releases/download/v26.06.106/HyperCapslock.dmg \
  --set-name hypercapslock

# 3) NEW-cask audit — note the flag shape
brew audit --new --cask hypercapslock
# 4) auto-fix style
brew style --fix hypercapslock

# 5) local install / uninstall / zap, bypassing the API
export HOMEBREW_NO_AUTO_UPDATE=1 HOMEBREW_NO_INSTALL_FROM_API=1
brew install   --cask hypercapslock
brew uninstall --cask hypercapslock
brew zap       --cask hypercapslock

# 6) PR from a fork of Homebrew/homebrew-cask
git checkout -b hypercapslock
git add Casks/h/hypercapslock.rb
git commit -v        # "Add HyperCapslock 26.06.106"
git push <your-fork> hypercapslock
