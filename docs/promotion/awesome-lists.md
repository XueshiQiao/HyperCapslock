# Awesome-list submissions (notability drivers)

Getting listed in well-trafficked "awesome" lists is the most reliable, evergreen
way to grow stars toward the Homebrew Cask notability bar (≥225 stars for an
author self-submission). Below are ready-to-paste entries + each list's rules.

Shared one-liner (keep consistent with the README tagline & cask `desc`):

> Remap Caps Lock to a system-wide vim-style navigation & editing layer via an
> F18 "Hyper" key — GUI, per-app rules, custom actions, no kernel extension.

---

## 1. jaywcjlove/awesome-mac  ⭐ highest traffic

- Repo: https://github.com/jaywcjlove/awesome-mac
- Section: **Productivity → Productivity Tools** (or **Keyboard**, whichever the
  current structure has — check before submitting).
- Rules ([CONTRIBUTING](https://github.com/jaywcjlove/awesome-mac/blob/master/CONTRIBUTING.md)):
  entries are alphabetical within a section and **must carry the license/price
  icons** via the repo's reference-style image links — append
  `![Open-Source Software][oss icon]` and `![Freeware][free icon]` exactly as the
  existing rows do (copy a neighboring line's icon refs).

Entry (adjust the icon refs to match the file's definitions):

```markdown
* [HyperCapslock](https://github.com/XueshiQiao/HyperCapslock) - Remap Caps Lock to a system-wide vim-style navigation & editing layer via an F18 Hyper key; GUI, per-app rules, no kernel extension. ![Open-Source Software][oss-icon] ![Freeware][free-icon]
```

PR title: `Add HyperCapslock`
PR body: one line — what it is + that it's open-source (GPL-3.0) and free.

---

## Before submitting any list

- Add a short **demo GIF** to the README (top, above the fold) — list reviewers
  and visitors convert far better with motion.
- Make sure the screenshots and tagline are current.
- Submit one list at a time; respect each repo's alphabetical ordering and icon
  conventions, or the PR gets bounced.
